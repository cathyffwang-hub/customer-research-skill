# 客户调研大纲生成 Skill

把**客户调研提问大纲**这件事交给 AI：你只说"我要调研 xx 客户 + 这是我的调研安排"，AI 帮你生成一篇腾讯文档智能文档格式的调研大纲。

## 功能

- 🎯 **两套模板自动切换**：根据客户"使用基础"（用得好 / 用得不好）选择对应的调研大纲结构
- ⏰ **场次自动排期**：支持单场次 / 多场次（每场一篇 or 合成一篇）
- 🔀 **产品视角自动切换**：腾讯文档 or 企微文档，全文表述自动统一
- 📝 **固定问题 + 灵活参数**：问题文案是业务侧定下来的固定模板，只替换部门/人员/时间等参数
- 📄 **一键建档到腾讯文档**：直接通过腾讯文档 MCP 生成 smartcanvas 格式在线文档
- 📦 **可选归档到知识库空间**（v1.6）：建完自动归档到「客户调研｜AI生成」/客户名子夹，同名空间/子夹自动复用不重建

## 安装

### 1. clone 到用户级 skills 目录

```bash
cd ~/.workbuddy/skills
git clone <此仓库地址> 客户调研大纲生成
```

### 2. 前置依赖

- 已安装并授权 **腾讯文档 skill**（本 skill 通过它调用 MCP 建档）
- 已安装 **mcporter** CLI
- Node.js 已就绪

### 3. 自动更新

每次 skill 被调起时，会自动执行 `scripts/update-check.js`：检测远端是否有新 commit，有则自动 `git pull` 拉取最新版。**你不需要手动 pull**，下次用就是最新版。

## 使用

在会话里直接说：

> 我要调研 XX 客户，这是我的调研安排：……

或：

> 帮我生成 XX 客户的企微文档场景调研大纲

AI 会根据 `SKILL.md` 定义的流程：

1. 询问缺失信息（产品版本、署名、使用基础、场次、背景目的、是否归档）
2. 读取对应模板
3. 替换参数
4. 调用腾讯文档 MCP 建档
5. （可选）归档到「客户调研｜AI生成」空间 / 客户名子夹
6. 返回文档 URL + 归档位置

## 目录结构

```
客户调研大纲生成/
├── SKILL.md                           # 入口：触发词 + 流程 + 硬约束
├── README.md                          # 你正在看的这个
├── scripts/
│   └── update-check.js                # Git 自动更新脚本
└── templates/
    ├── template_A.mdx                 # 场景挖掘型（客户用得不好）
    ├── template_A_session.mdx         # 模板 A 的单场次块
    └── template_B.mdx                 # 最佳实践型（客户用得好）
```

## 修改规则

- **正式调研部分的问题文案不要改**：这是业务方定的模板，只替换参数
- **新增字段**：在 SKILL.md 的"参数替换规则"表里追加，并在模板里用 `{{xxx}}` 占位
- **新增场景分支**：在 templates/ 下再加一个模板文件，并在 SKILL.md 里加分支判断说明
